# Contributing to Studio Video Editor Multicam

Thanks for contributing.

## Workflow
1. Create a feature/fix branch from the default branch.
2. Keep changes small and focused.
3. Run local checks before opening a PR:
   - `npm install`
   - `npm run build`
   - `npm test` (if available)
4. Update docs/changelog when behavior changes.

## Commit style
Use clear commit messages, e.g.:
- `docs: update setup instructions`
- `fix: handle invalid input in converter`
- `chore: update dependencies`

## Pull requests
Include:
- What changed
- Why it changed
- How it was tested
- Any screenshots for UI updates
